# Створювач Навичок (Skill Creator)

Ця навичка містить інструкції щодо створення ефективних навичок.

## Про Навички (Skills)

Навички — це модульні, самодостатні пакети, які розширюють можливості Codex/агента, надаючи йому спеціалізовані знання, робочі процеси та інструменти. Сприймайте їх як "посібники з адаптації" для конкретних доменів чи завдань — вони перетворюють агента загального призначення на спеціалізованого фахівця, який володіє процедурними знаннями, доступними далеко не кожній моделі за замовчуванням.

### Що забезпечують Навички
* **Спеціалізовані робочі процеси** - Багатокрокові процедури для окремих доменів.
* **Інтеграції з інструментами** - Інструкції для роботи з конкретними форматами файлів або API.
* **Доменну експертизу** - Специфічні для компанії знання, схеми (schemas), бізнес-логіку.
* **Пакетні ресурси** - Скрипти, документацію (references) та активи (assets) для виконання складних або повторюваних завдань.

## Основні Принципи

### Лаконічність — це Головне
Контекстне вікно — це спільний ресурс. Навички ділять його з усім іншим, що потрібне агенту: системним промптом, історією чату, метаданими інших навичок та власне запитом користувача.

Базове припущення: агент вже достатньо "розумний". Додавайте ЛИШЕ той контекст, якого він однозначно не знає. Ставте під сумнів кожну інформацію: "Чи дійсно агенту потрібно це пояснювати?" та "Чи виправдовує цей абзац свою вартість у токенах?"

Надавайте перевагу лаконічним прикладам перед довгими теоретичними поясненнями.

### Визначте Відповідний Ступінь Свободи
Зіставте рівень деталізації з тим, наскільки задача є варіативною або схильною до збоїв:
* **Високий ступінь свободи (текстові інструкції):** Використовуйте, коли допустимі кілька варіантів вирішення, рішення залежать від контексту, або коли підхід ґрунтується на евристиках.
* **Середній ступінь свободи (псевдокод або скрипти з аргументами):** Використовуйте, коли існує єдиний бажаний патерн, але з допустимими варіаціями або коли конфігурація впливає на поведінку.
* **Низький ступінь свободи (специфічні скрипти з мінімумом параметрів):** Використовуйте для критичних та схильних до помилок операцій, коли потрібна абсолютна консистентність і дотримання конкретної послідовності.

## Структура Навички

Кожна навичка складається з обов'язкового файлу `SKILL.md` та додаткових ресурсів (packaged resources):

```
skill-name/
├── SKILL.md (обов'язково)
│   ├── YAML-заголовок (metadata) (обов'язково)
│   │   ├── name: (обов'язково)
│   │   └── description: (обов'язково)
│   └── Markdown-інструкції (обов'язково)
└── Додаткові ресурси (опціонально)
    ├── scripts/          - Виконуваний код (Python/Bash тощо)
    ├── references/       - Документація, яка завантажується в контекст лише за необхідності
    └── assets/           - Файли для використання в кінцевому результаті (шаблони, іконки, тощо)
```

### SKILL.md (обов'язково)
Кожен файл `SKILL.md` складається з:
* **Заголовка (YAML):** Містить поля імені (`name`) та опису (`description`). Це єдині поля, які спочатку "бачить" агент, щоб визначити, чи слід застосувати цю навичку до поточного завдання.
* **Тіла інструкцій (Markdown):** Безпосередньо інструкції та вказівки. Цей блок завантажується в контекст ЛИШЕ ПІСЛЯ спрацювання навички.

### Додаткові Ресурси (опціонально)

#### Скрипти (scripts/)
Виконуваний код для виконання завдань, що вимагають абсолютної детермінованості (100% надійності без помилок генерації).
* **Коли включати:** Коли той самий код потрібно писати/згадувати постійно, або коли без права на помилку необхідний детермінований результат.
* **Приклад:** `scripts/rotate_pdf.py`

#### Документація / Довідники (references/)
Матеріали, які повинні завантажуватися лише тоді, коли виникає конкретна потреба.
* **Коли включати:** Для технічної документації, до якої потрібно звертатися під час роботи (наприклад, специфікації API, правила компанії, структури бази даних).
* **Перевага:** Дозволяє тримати основний файл `SKILL.md` максимально чистим та невеликим.

#### Активи (assets/)
Будь-які файли, які можуть бути використані агентом у його кінцевому результаті (наприклад: логотипи, стартові шаблони для проектів, boilerplates).

## Принцип Поступового Завантаження 
Система застосовує трирівневе завантаження навичок:
1. **Метадані (name + description)** - Вони завжди в контексті (~100 слів).
2. **Тіло SKILL.md** - Завантажується, коли навичка була викликана агентом або користувачем (< 5000 слів).
3. **Пакетні ресурси** - Завантажуються за окремими запитами у міру нагальної потреби (Необмежено).

Утримуйте кількість рядків у файлі `SKILL.md` нижче 500, щоб не перевантажувати контекстне вікно.

## Процес Створення Навички

1. **Зрозумійте суть навички** завдяки конкретним прикладам.
2. **Сплануйте ресурси**, необхідні для навички (скрипти, довідники, активи).
3. **Ініціалізуйте навичку** (бажано через `init_skill.py`).
4. **Наповніть її контентом** (імплементуйте скрипти та напишіть `SKILL.md`).
5. **Запакуйте навичку** (через команду `package_skill.py`).
6. **Ітеруйте та покращуйте** її на основі реального використання.

### Крок 3: Ініціалізація навички (Якщо використовується CLI-інструмент)
Приклад:
`scripts/init_skill.py <skill-name> --path <output-directory> [--resources scripts,references,assets] [--examples]`

### Крок 5: Пакування навички
Приклад:
`scripts/package_skill.py <path/to/skill-folder>`

Скрипт перевіряє валідність навички і створює архів з розширенням `.skill` для дистрибуції.
